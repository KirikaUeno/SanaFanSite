The description of the ideal system, problems and plans for the object edition system, as well as the current progress and realization details

All the games now offer to the player a super-restricted system of craft: only the objects that were created by the developers can be crafted and used.
One of the main principles of this project is to make as few rules as possible, and the restricted craft system is a major rule. Moreover, the game is about civilization progress, one of the main idea of which is that players have to come up with reciepts with themselves, and they will surely find a lot of working in the real world possibilities (additional reciepts, other ways to craft the same item and so on), that developers did not take into account.

Obviously, the fully free system of craft is not possible now - it requires a full stack of fields of physics with electrodynamics, fluids dynamics etc, so I am trying to create the easy-mesh-changer as a first step. As a first step of this first step, this system should allow player to cut an item.
The main problem for now is to create a hole. I want to gradually 'drill' a deeping in the object. This can be done in several ways, but some of them doesnt allow the intersection with the opposite surface. Merge these surfases is the task for now.
